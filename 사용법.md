# 딩고 페이지 SVG 이미지 분리 — 사용법

`딩고 페이지.svg` 안에 base64로 박혀 있는 **이미지 18개**를 `images` 폴더로 빼내고,
SVG 안에서는 이미지를 **파일 경로로 연결**해주는 도구입니다.
결과 SVG는 원본과 **똑같이 보이며**, 나중에 `images` 폴더의 파일만 바꾸면 디자인에 그대로 반영됩니다.

## 실행 방법 (둘 중 하나)

1. **`실행하기_더블클릭.bat`** 파일을 더블클릭 — 가장 쉬움 (파이썬 설치 필요)
2. 또는 폴더에서 명령창을 열고: `python "svg_이미지_분리.py"`

> 파이썬이 없다면 https://www.python.org/downloads/ 에서 설치하세요.
> 설치할 때 **"Add Python to PATH"** 를 꼭 체크하세요.

## 실행하면 생기는 것

- `images/` — 추출된 이미지들 (`딩고_image0_814_1742.png` 형식)
- `딩고 페이지_경로버전.svg` — 이미지가 경로로 연결된 새 SVG (원본은 그대로 보존)

## 옵션 (svg_이미지_분리.py 위쪽)

- `EXTRACT = True` — 원본 이미지를 실제 파일로 뽑아서 연결 (기본, 원본과 100% 동일하게 보임)
- `EXTRACT = False` — 이미지를 뽑지 않고 **빈 경로만** 만들어 둠 (이미지를 처음부터 직접 넣고 싶을 때)

## 이미지 교체

`images` 폴더에서 바꾸고 싶은 이미지를 **같은 파일 이름**으로 덮어쓰면 됩니다.
파일 이름을 바꾸고 싶으면 `딩고 페이지_경로버전.svg` 안의 `href="images/..."` 경로도 함께 수정하세요.

## 참고

원래는 제가 직접 변환해 드리려 했지만, 이 파일의 이미지 데이터가 매우 커서
제 쪽 코드 실행 환경(임시 리눅스)이 디스크 공간 부족으로 작동하지 않았습니다.
그 환경이 복구되면 제가 직접 변환해 드릴 수도 있으니 알려주세요.
